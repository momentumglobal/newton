Frequently Asked Questions

Who can access the Market Reporting module?
Admins, Delivery Managers, and Talent Partners. Leadership and Viewer roles do not have access.
Why can I only see certain roles in the Role dropdown?
The roles available to you depend on your access level. Talent Partners see only roles they are directly assigned to. Delivery Managers see all roles across their assigned projects. Admins see all roles in the system.
Where does the data in the report come from?
All figures are drawn from the Weekly Activity SharePoint list. Outreach, Responses, Screened, Submitted, Interview stages, Offers, and Hires are summed across all activity records logged against the selected role — there is no date filter. The report always shows all-time totals for that role.
What is the Total Addressable Market (TAM)?
TAM is the estimated total pool of relevant candidates for the role. You set this manually when building the report. It is used to calculate the % of TAM Contacted and % of TAM Responded KPIs, giving stakeholders context for how far into the market you have reached.
The report title was autogenerated but I want to change it — can I?
Yes. The title field is fully editable. Once you type in it manually, the autogenerate will not overwrite it again for that session.
Can I see saved reports created by other users?
Talent Partners and Delivery Managers can only see reports they created themselves. Admins can see all saved reports across the team.
How often does the data update?
Newton reads directly from SharePoint via the Microsoft Graph API. Clicking Generate Report always fetches the latest data — there is no cache to clear.
Who do I contact if something isn't working?
Raise it with your Newton Admin. For access issues, they can update user assignments via the Config Panel in the Reporting module.

User Roles

Three roles can access Market Reporting, each with different visibility over which roles they can report on.

Admin

Full access. Can generate reports for any role in the system. Can view, edit, and delete all saved reports regardless of who created them.

Delivery Manager

Can generate reports for all roles across their assigned project(s). The Project dropdown is visible to filter the Role list. Can view and manage their own saved reports.

Talent Partner

Can generate reports only for roles where they are the assigned Talent Partner. No Project dropdown — the Role dropdown is pre-scoped to their assignments. Can view and manage their own saved reports.

Exporting to PDF

Click ⊘ Export PDF to open the browser print dialog. Configure it as follows before saving:

Destination

Set to Save as PDF.

Options

Ensure Headers and footers is unticked and Background graphics is ticked. Background graphics must be enabled for the branded header block and table styling to print correctly.

These settings are not saved by the browser between sessions — you will need to set them each time you export.

Market Report Builder

Admin Delivery Manager Talent Partner

The Market Report Builder generates a fixed-layout client-facing report for a single role, summarising sourcing reach, pipeline activity, and your written observations.

Building a report

Complete the fields in the left-hand panel, then click Generate Report:

Report Title — autogenerated from your project and role selections in the format Customer x Momentum - Role (Month Year). Edit freely at any time.

Project (Admin and DM only) — select the customer project to filter the Role dropdown.

Role — select the specific role you are reporting on. Only roles within your access scope are shown.

Total Addressable Market (TAM) — enter the estimated candidate pool size for this role. This drives the two TAM percentage KPIs.

Report layout

Once generated, the report displays four sections in a fixed order: KPI Strip, Resulting Pipeline Activity, Offer Rejection Reasons (only shown if rejection data exists for the role), and Observations & Recommendations.

Action buttons

Saved Reports — open the list of reports you have previously saved.

Save — save the current report to SharePoint. Saving again updates the existing record rather than creating a duplicate.

Preview — open a read-only preview of the report as it will appear when exported.

⊘ Export PDF — render the report for printing and open the browser print dialog.

Always click Generate Report before saving or exporting. The save and export functions work from the most recently generated data.

KPI Strip

Admin Delivery Manager Talent Partner

Four headline figures displayed as tiles at the top of the report. All figures are all-time totals for the selected role — there is no period filter.

Total Addressable Market

The TAM value you entered in the left-hand panel.

% of TAM Contacted

Total Outreach ÷ TAM × 100. Shows how far into the candidate market you have reached with direct outreach.

% of TAM Responded

Total Responses ÷ TAM × 100. Shows the proportion of the market that has engaged with your outreach.

Days Open

The number of calendar days since the role's Open Date was set, calculated automatically from the Roles list.

If Days Open shows "—", the Open Date has not been set on the role record. Update it in the Roles page within the Reporting module.

Resulting Pipeline Activity

Admin Delivery Manager Talent Partner

A single-row summary table showing cumulative pipeline totals for the role across all time. Columns: Screened, Submitted, IV1, IV2+, Final IV, Offers, Hires.

Figures are summed across every Weekly Activity record associated with the role — not filtered by period or Talent Partner.

Offer Rejection Reasons

If any rejected offers have been logged against this role, a second tile appears below the pipeline table listing each rejection with candidate name, reason, and any additional detail. If there are no rejections recorded, this tile is hidden automatically.

Observations & Recommendations

Admin Delivery Manager Talent Partner

A free-text editor for your written commentary on the market. Use it to summarise what the data means, highlight trends, and set out recommended next steps for the client.

Formatting toolbar

The toolbar above the editor provides: Bold, Italic, Underline, bullet list, numbered list, Heading (larger bold text with a rule), and Body Text (returns a heading back to normal paragraph style).

Write your observations before saving or exporting — the content of this editor is included in both the Preview and the PDF export.

Saved Reports

Admin Delivery Manager Talent Partner

Click Saved Reports in the top-right to open the list of previously saved reports. Each entry shows the report title, role name, and the name of the user who created it.

Opening a report

Click Open to load a saved report. The builder repopulates with the saved title, role, TAM, and observations, and the report is automatically regenerated with the latest live data from SharePoint.

Editing a saved report

Click Edit to load the report into the builder. Make your changes, then click Save to update the record.

Deleting a saved report

Click Delete and confirm the prompt. Deletion is permanent within Newton — use SharePoint's recycle bin if you need to recover a deleted item.

Talent Partners and Delivery Managers can only see and manage reports they created themselves. Admins can see, edit, and delete all saved reports across the team.
