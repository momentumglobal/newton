Frequently Asked Questions

Who can access the People module?
The People module is restricted to Admin and Leadership roles. Delivery Managers and Talent Partners do not have access.
How does utilisation get calculated?
Utilisation is calculated as total billed capacity divided by total available capacity across all active employees (excluding CSD level). Each monthly row is prorated based on how many calendar days of an assignment fall within that month.
Why do the dashboard figures only show up to today?
All utilisation and revenue calculations are capped at today's date. Even if an assignment runs into the future, only the days up to and including today are counted. This gives an accurate picture of what has actually been delivered.
Why does an employee appear in multiple customer groups on the Deployment Timeline?
An employee can have assignments with more than one customer. Each assignment is grouped under its own customer, so the same employee may appear in multiple sections of the timeline if they have worked across customers during the selected year.
What does "Estimated Revenue" mean?
Revenue figures are calculated from monthly bill rates and assignment dates. They are estimates based on signed assignment data — actual invoiced amounts may differ slightly. G-P Invoice amounts are tracked separately in the G-P Invoices page.
How often does the data update?
Newton reads directly from SharePoint Lists via the Microsoft Graph API. Data updates in real time — refreshing the page will reflect the latest entries.
Who do I contact if something isn't working?
Raise it with your Newton Admin. For access issues, they can update user assignments directly via the Admin link on the Newton home screen.

User Roles

The People module is accessible to two roles only.

Admin

Full access to all People module pages. Can add, edit, and delete employees, assignments, and invoices. Sees all data across the entire workforce.

Leadership

Read-only access to all People module pages. Can view the dashboard, employee list, deployment timeline, and invoices but cannot add or edit any records.

Exporting to PDF

Pages with an ⎙ Export PDF button can be saved as a PDF document. Click the button to open your browser's print dialog, then configure it as follows before saving:

Destination

Set to Save as PDF. This saves the output as a file rather than sending it to a physical printer.

Options

Under the Options section, ensure Headers and footers is unticked and Background graphics is ticked. Background graphics must be enabled for the branded header block to print correctly.

These settings are not saved by the browser between sessions — you will need to set them each time you export.

People Dashboard

Admin Leadership

A real-time overview of workforce performance, utilisation, and revenue. The KPI strip is always fixed to current year YTD — the period filters below it control the three detail panels only.

KPI Strip

Four tiles showing company-wide metrics as of today:

Estimated Revenue — total billed revenue for the current year to date.

Utilisation — billed capacity as a percentage of total capacity, YTD. Colour coded: green = 85%+, orange = 75–85%, red = below 75%.

Active Customers — number of distinct customers with a current active assignment today. The delta indicator (green/red) shows the change vs the last day of the previous quarter.

Billed Headcount — number of employees on a billed assignment today. The delta indicator shows the change vs the last day of the previous quarter.

Period Filters

Use the Year, Month, and Quarter dropdowns to filter the three panels below. Selecting a month clears the quarter selection and vice versa. The year filter applies to both.

Panel 1 — Team Utilisation

A summary table showing utilisation percentage and headcount (utilised / total active) per role band — SDM, STP, TP. A totals row shows the overall blended utilisation. Below the table, a monthly trend bar chart shows how utilisation has moved across the selected period.

Panel 2 — Estimated Revenue

Two side-by-side tables for the selected period: revenue by customer (sorted highest to lowest, excluding Unassigned/Internal) and revenue by project type (Embedded, CoE, Transformation, LCI).

Panel 3 — Workforce Segmentation

Three headcount breakdowns of currently active employees (not period-filtered): by location, by contract type, and by role band. Each shows count and percentage of total.

Upcoming End Dates

Lists any active employees with an end date in the next 60 days, sorted by nearest first.

Employee Tracker

Admin Leadership

A two-tab view of the workforce — employees and their assignments. Use the tab bar at the top to switch between the two views.

Employees tab

Lists all active employees sorted by role band then name, showing level, contract type, location, start date, end date, and status. Toggle Show inactive employees to include leavers in the list.

Admins can add new employees using + Add Employee and edit existing records via the Edit link on each row.

An employee's Level field (SDM, STP, TP, CSD) determines which role band they appear in across all dashboard calculations. Keep this accurate when editing records.

Assignments tab

Lists assignment records showing employee name, level, customer, project type, start and end dates, monthly bill rate, and billed status. Defaults to current assignments.

Use the Status filter to switch between Current, Former, Planned, or All assignments. Filter further by Customer or Project Type using the dropdowns.

Admins can add new assignments using + Add Assignment and edit existing records via the Edit link on each row.

The Billed field on an assignment determines whether it contributes to revenue and utilisation calculations. An assignment marked Billed = No counts toward capacity but not billed capacity.

Deployment Timeline

Admin Leadership

A read-only Gantt-style view of all employee assignments across a calendar year. Use the Year selector at the top right to switch between years.

Reading the timeline

Rows are grouped by customer (A–Z). Within each customer, employees are sorted by role band (SDM, STP, TP) then name. Unassigned or bench employees appear at the top of the chart. Each bar represents one assignment — hover over a bar to see the customer, bill rate, and assignment dates in a tooltip.

Bars are colour coded by project type:

Embedded
CoE
Transformation
LCI
Internal / bench

Multiple assignments

If an employee has more than one assignment in the year, all bars appear on the same row. Bars from different assignments may overlap if the employee was deployed across more than one customer simultaneously.

The Deployment Timeline shows future assignment dates as planned — it is not capped at today. This makes it useful for workforce planning and visualising upcoming capacity changes.

G-P Invoices

Admin Leadership

Tracks invoices issued through the G-P (Employer of Record) arrangement. Each invoice is logged with its number, dates, amount, notes, and payment status.

Summary bar

At the top of the page, three figures give an at-a-glance view of the invoice position: total outstanding amount, number of overdue invoices, and the date of the oldest overdue invoice.

Invoice status

Each invoice has one of three statuses: Sent (issued, awaiting payment), Overdue (Sent with a due date in the past), or Paid. Overdue status is calculated automatically — no manual flagging is needed.

Adding and editing invoices

Admins can add new invoices using + Add Invoice and edit existing records via the Edit link on each row. Once payment is received, use Mark Paid on the row to update the status directly without opening the edit form.

G-P Invoice amounts are tracked separately from the Estimated Revenue figures on the People Dashboard. The dashboard calculates revenue from assignment bill rates — invoice amounts here reflect actual G-P charges which may differ.
