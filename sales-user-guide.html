Frequently Asked Questions

Who can access the Sales module?
Admin and Leadership have full access to all Sales pages. Delivery Managers can access the module but are limited to the LCI Cost Models page — they land there automatically on sign-in and see only the models assigned to them. Talent Partners do not have access.
What is an LCI Cost Model?
Location & Cost Intelligence (LCI) models are the recruitment ramp & cost models we build on location-research engagements — a client wants to build or transition a team into a new location, and we model the salary spend, recruitment fees, and any crossover costs over time. The LCI Cost Models page replaces the Excel version we used to build by hand.
Why does a model have two currencies?
Salaries are scoped in the local currency of the new location (e.g. Romanian Leu), but the client usually wants the model presented in their own currency (e.g. Euro). Each model has a Local currency for the CoE-side inputs and a Display currency for everything the client sees; a manual FX rate converts between them. If both are the same, the FX rate is ignored.
Are the salary figures annual or monthly?
Salaries are entered as annual amounts. Newton splits them into a monthly cost automatically, adding employer burden, any bonus %, and a 13th/14th-month gross-up if the market uses one.
When does a hire start costing money?
A hire placed in a given month starts hitting payroll after the notice period set in the model's settings. For example, a hire in M2 with a one-month notice period starts costing from M3. Starts are assumed to be on the 1st of the month.
If I save a report and a model changes later, does the report update?
Yes. A saved report stores which models it contains plus your observations text — not the numbers. When you re-open it, the figures are recalculated from the current model data. The PDF you exported at the time stays as the fixed record of what was presented.
How often does the data update?
Newton reads directly from SharePoint Lists via the Microsoft Graph API. Data updates in real time — refreshing the page reflects the latest entries.
Who do I contact if something isn't working?
Raise it with your Newton Admin. For access issues (including which models are assigned to you), they can update the model's Assigned DM directly.

User Roles

The Sales module is accessible to three roles, with different reach.

Admin

Full access to every Sales page — Revenue Tracking, Sales Forecast, and LCI Cost Models. Can create, edit, copy and delete any model, assign models to Delivery Managers, and delete saved reports.

Leadership

Full access to all Sales pages and all LCI models and reports. Can create and edit models and reports, but cannot delete models (Admin only).

Delivery Manager

Access to the LCI Cost Models page only — Delivery Managers land there directly on opening the Sales module. They see and edit only the models assigned to them (matched on their email), and only their own saved reports. Revenue Tracking and Sales Forecast are not visible to this role.

Exporting to PDF

Model summaries and reports are saved as PDFs through your browser's print dialog. Click Print / PDF to open it, then configure as follows before saving:

Destination

Set to Save as PDF.

Layout & Options

Use Landscape layout, untick Headers and footers, and tick Background graphics — background graphics must be on for the navy cover pages, table shading and charts to print correctly.

These settings are not saved by the browser between sessions — set them each time you export. For the best result use Chrome or Edge; the branded cover/divider pages use print features that are most reliable in Chromium browsers.

Revenue Tracking

The default landing page for Admin and Leadership. A line graph of estimated monthly revenue across the year, with green / amber / red threshold bands behind the line so you can see at a glance which months are healthy.

Estimated vs Forecast

The solid line is estimated revenue, derived from live assignment data (each assignment's monthly bill rate, pro-rated by how many days fall in each month). The dashed orange line is estimated + forecast, forking at the current month and using the pipeline's forecast headcount and revenue-per-head.

Year selector

The year dropdown lists only the years your assignment data spans. Switching year redraws both lines and the threshold bands.

Sales Forecast

A simple pipeline table of forecast engagements — customer/project, start and end dates, forecast headcount, an optional monthly revenue-per-head figure, and notes. Use + Add Forecast to add a row, or Edit / Delete on an existing one.

Forecast rows feed two places elsewhere in Newton: the dashed forecast line on Revenue Tracking, and the Command Centre's Revenue and Utilisation tiles.

LCI Cost Models — Overview

The LCI Cost Models page is the home for Location & Cost Intelligence models. It opens on a list of your models with quick filters for Client and Location, and actions to create, open, edit, copy, compare, report on, and (for Admin) delete them.

Model status

Every model carries a status, shown as a coloured pill:

Draft Work in progress.
Presented Shared with the client.
Won Engagement confirmed — a Won model can be linked to a delivery project (see Linking to a Hiring Plan).
Lost Not proceeding.

Row actions

  • Open — open the model editor.
  • Edit — quick-edit the header details (name, client, location, currencies, start month, horizon, assigned DM) without opening the full editor.
  • Copy — duplicate a model (all rows and milestones), reset to Draft. Ideal for building a v2 or a second location from an existing model.
  • Link / Plan ✓ — (Won models only) link to a CoE delivery project.
  • Delete — Admin only; removes the model and all its rows.

Creating a Model

Click + New Model and complete the dialog: model name, client name, CoE location, the local and display currencies (with an FX rate if they differ), the start month (this becomes M1), and the horizon in months. Leadership and Admin can also assign the model to a Delivery Manager here.

A Delivery Manager who creates a model is automatically assigned to it, so it appears in their list straight away.

The model is created as a Draft. Click Open to start building it in the editor.

Settings & Currencies

The settings bar at the top of the editor holds every assumption behind the model. Changing a setting and clicking Save Settings instantly recalculates the whole model.

Currencies

The Local currency applies to the CoE-side inputs (role salaries and office cost per head). The Display currency applies to the customer-side inputs (legacy team, retention/relocation, project fees) and to every output — tables, charts, KPIs and the exported report. When the two differ, enter an FX rate (local → display); it is hidden when they match.

EoR fee and Travel are the two exceptions to the "CoE side = local currency" rule — they are entered in the Display currency, because they are typically billed to the client in their own currency.

Cost assumptions

  • Employer burden % — added on top of salary + bonus for every CoE and legacy role.
  • Salary months — 12, 13 or 14, to gross up salaries in markets with a 13th/14th-month convention (e.g. Spain, Portugal).
  • Notice period (months) — delays the point at which a hire starts costing money (a hire in M2 with a 1-month notice starts on payroll in M3).
  • Office / EoR / Travel per head — monthly per-head running costs.

Section toggles

Not every engagement needs every cost section. The toggles let you switch off Legacy Team, Retention & Relocation, or Project Fees for a given model, and they disappear from the editor and the output.

Hiring Roadmap

The roadmap is where you plan the CoE hires month by month. Each row is a role (grouped by team); enter the annual salary, bonus %, and the number of hires in each month. The Hires and Cost/mo columns update live, along with the Hires-per-month and Cumulative-hires totals at the foot.

Repeat hires are easy — a single "Software Engineer" row can carry, say, 3 hires in one month and 2 in another. You don't need one row per person.

Project milestones

Milestones (e.g. Project Signoff, Offers Made, Employees Start) sit as their own rows at the top of the roadmap, with a bar spanning the months they cover — aligned to the same month columns as the hires, so the plan reads like the Excel version. Use + Add Milestone to add one.

Salaries here are shown in the model's local currency. Click Save Roadmap to save both the hires and milestones.

Cost Sections

Below the roadmap are the remaining cost inputs, all entered in the model's display currency:

Legacy Team

The existing team being replaced during a transition. Enter each role with its annual salary, quantity, and an exit month — the cost runs from M1 up to and including the exit month, then stops. A warning marker (⚠) appears if no exit month is set (the cost would otherwise run to the end of the horizon).

Retention & Relocation

One-off amounts (retention packages, relocation costs) entered in the specific month(s) they land.

Project Fees

Momentum's own fees — build team, licences, office sourcing and so on — entered as monthly amounts. Each fee line is itemised in the cost output with a subtotal.

Cost Model & Chart

At the bottom of the editor, a read-only Cost Model table mirrors the classic spreadsheet layout: per-team employee cost, CoE operating costs (EoR, office, travel), legacy costs, one-offs, project fees, total monthly spend and cumulative spend — all in the display currency, all recalculating live as you edit above.

Below the table, a Cumulative Spend chart plots the running total across the horizon.

Headcount rows show the number of people on payroll in each month, which lags the recruitment plan by the notice period. The roadmap's "hires per month" is recruitment-based; the cost model's headcount is payroll-based.

Salary Benchmarks

As you type a role title in the roadmap, Newton may show a small hint under the salary box, e.g. Benchmark: 34,000 (n=5). This is the median salary for that exact role title in the same location and local currency, drawn from all your other models. Click apply to drop the median into the salary cell.

The benchmark uses every prior model equally, and grows more useful as the library of models builds up. If there's no match for the title/location/currency, no hint appears.

Consistent naming helps — "Senior Software Engineer" and "Snr Software Engineer" are treated as different titles, and locations must match too. Keeping titles and location names consistent across models makes the benchmarks far more valuable.

Comparing Models

To compare locations or versions, tick two or more models on the list page (they must share the same display currency) and click Compare. The comparison shows a side-by-side KPI table — total spend, steady-state monthly and annual cost, cost per head, total hires, time to full ramp, and peak crossover spend — with a delta column when exactly two models are selected. A single cumulative-spend chart overlays all the selected models.

Peak crossover spend is the highest combined monthly cost during the transition — the worst month of paying both the new CoE team and the legacy team at once. A useful figure when discussing transition budgets with a client.

Summary & Reports

Single-model summary

From inside the editor, Summary / Print opens a clean, read-only view of one model — hiring plan with milestones, cost model, cumulative spend chart, and an assumptions page — ready to print or save as a PDF for the client.

Full report export

From the model list, tick one or more models and click Export Report. Newton assembles a branded, multi-page document: a navy cover page, a section per model, a Location Comparison section (when two or more models share a currency), and a final Observations and Recommendations page with a rich-text editor for your narrative. Then Print / PDF to produce the client deliverable.

Saving a report

Click Save Report to keep a report for later. Saved reports appear in the Saved Reports table on the list page. A saved report stores its title, the models it contains, and your observations text — the figures are always recomputed live when you re-open it, so a saved report never goes stale. Use Edit selection from within a report to add or remove models and re-save.

Admin and Leadership see all saved reports; Delivery Managers see only the ones they created. Deleting a report never affects the underlying models.

Linking to a Hiring Plan

Once an engagement is Won, the model can hand over to delivery. Click Link on a Won model, choose the CoE project, and either just save the link or Save & Generate Plan.

Generating creates one row in the project's CoE Hiring Plan (in the Reporting module) for every hire in the roadmap, with each row's open date worked backwards from its hire month so recruitment lands in the right month. Talent Partners are left blank for the Delivery Manager to assign.

This is a one-time hand-off: after generating, the Hiring Plan is the live delivery record and further edits to the model don't change it. If the project already has plan rows, Newton asks before adding more. Phase lengths use the Hiring Plan's own defaults, not the model's notice period.
