Frequently Asked Questions

How do I sign in?
Click "Sign in with Microsoft" on the Newton login page and authenticate with your Momentum Global Microsoft account. No separate password is required.
Why can't I see all pages in the navigation?
The pages you see depend on your assigned role. Talent Partners see project-level pages; Delivery Managers see the same plus additional detail panels; Leadership sees Company Dashboard; Admins see everything including the Admin Panel.
I can't see my project — what do I do?
Your project access is controlled by your user assignment in the Admin Panel. Contact your Admin to ensure you've been assigned to the correct project.
Why are some fields showing "—"?
A dash means the data hasn't been entered yet for that record. Fields like Talent Partner, Target Hire Date, and Budget are optional — they'll show "—" until populated.
How often does the data update?
Newton reads directly from SharePoint Lists via the Microsoft Graph API. Data updates in real time — refreshing the page will show the latest entries.
What does the KPI Period filter control?
The Month / Quarter / Year buttons on the dashboards filter the pipeline metrics (submissions, interviews, offers, hires) to activity recorded within that period of the current calendar year.
Who do I contact if something isn't working?
Raise it with your Newton Admin. For access issues, they can update user assignments directly in the Admin Panel.

User Roles

Every Newton user is assigned one of four roles. Your role controls which pages you can see and whether you can add or edit data.

Admin

Full access to all pages and all projects. Can add, edit, and manage all records. Has access to the Admin Panel for user and project management.

Delivery Manager

Access to Project Dashboard, Projects, Roles, Placements, Rejected Offers, and Weekly Activity for their assigned projects. Can add and edit records. Sees all Talent Partner activity within their projects.

Talent Partner

Access to Project Dashboard, Roles, Weekly Activity, Placements, and Rejected Offers scoped to their assigned project and their own activity. Can add and edit records.

Leadership

Read-only access to Company Dashboard. Provides a cross-project view of open roles, pipeline metrics, and Talent Partner activity. Cannot add or edit records.

Company Dashboard

Admin Leadership

A cross-project rollup giving a company-wide view of recruitment health. Data is not filtered by project — it covers all active projects.

KPI Tiles

The top strip shows current Open Roles and Active Projects. The second strip shows pipeline metrics for the selected period: Hires, Submission Conversion, IV to Offer Ratio, Offer Success, Average Days to Hire, and Hired On Time percentage. Use the Month / Quarter / Year buttons to change the period.

Roles Open 30+ Days

Lists all active roles that have been open for 30 or more days, sorted by project. Rows are colour-coded: pale orange = 30–44 days open; pale red = 45+ days open.

Activity by Talent Partner

Shows outreach, submissions, interviews, offers, and hires per Talent Partner per project for the selected period. Use the Period dropdown to change the date range.

Project Dashboard

Admin Delivery Manager Talent Partner

A detailed view of recruitment performance for a single project. Delivery Managers and Admins can switch between projects using the Project dropdown. Talent Partners see their assigned project only, scoped to their own activity.

KPI Tiles

Shows Open Roles (current) and Hires to Date (all time), plus period metrics: Submission Conversion, IV to Offer Ratio, Offer Success, Average Days to Hire, and Hired On Time. Use Month / Quarter / Year to change the KPI period.

Roles Open 30+ Days

Same colour-coded table as the Company Dashboard, filtered to the selected project only. No Project column — Role, Talent Partner, Stage, and Days Open are shown.

Pipeline Activity

A breakdown of weekly activity by role for the selected period, including Outreach, Responses, Screened, Submitted, Interview stages, Offers, and Hires. A Totals row summarises across all roles.

Activity by Talent Partner (DM / Admin only)

Aggregates pipeline activity by Talent Partner for the selected period.

Offer Rejection Reasons (DM / Admin only)

Counts rejected offers by reason category for the period.

Upcoming Starters (DM / Admin only)

Lists placed candidates with a provisional start date in the future.

Actual Spend vs Budget (DM / Admin only)

Compares total agreed salaries against total role budgets, showing the variance as over or under budget.

Use the Period dropdown to filter the detail panels independently from the KPI period buttons.

Projects

Admin Delivery Manager Talent Partner

Lists all projects you have access to, showing the customer name, status (Active / Transition / Completed), and start date. Admins and Delivery Managers can add new projects and edit existing ones using the buttons in the table.

Project status drives what appears in the Company Dashboard — only Active projects count toward the Active Projects KPI.

Roles

Admin Delivery Manager Talent Partner

A list of all roles across your accessible projects. Each row shows the role title, project, stage, assigned Talent Partner, open date, and target hire date.

Filtering

Use the Project dropdown to filter by project. Use the stage filter buttons (All / Active / Backlog / On-hold / Hired / Cancelled) to narrow by stage.

Adding and editing roles

Admins and Delivery Managers can add new roles using the + Add Role button. Any role can be edited or its stage updated using the action links in the table.

The "Roles Open 30+ Days" panel on both dashboards is driven by the Open Date field — make sure this is set accurately when a role is created.

Weekly Activity

Admin Delivery Manager Talent Partner

Records weekly pipeline activity against specific roles — Outreach, Responses, Screened, Submitted, Interview 1, Interview 2+, Final Interview, Offers, and Hires. Each entry is tied to a role, a Talent Partner, a year, and a week number.

Filtering

Filter by project using the dropdown. Talent Partners see only their own entries; Delivery Managers and Admins see all entries for their projects.

Adding activity

Use + Add Activity to log a week's numbers. Select the role, enter the week ending date, and fill in the activity counts. This data feeds directly into the dashboard pipeline panels.

Placements

Admin Delivery Manager Talent Partner

Records successful hires — candidate name, role, agreed salary, and provisional start date. Each placement is linked to a specific role.

Filtering

Filter by project, and by Month, Quarter, or Year using the filter rows below the project dropdown.

Recording a placement

Use + Record Placement to log a hire. The salary field feeds the Actual Spend vs Budget panel on the Project Dashboard, so accuracy here matters.

A placement's provisional start date is what populates the Upcoming Starters panel on the Project Dashboard.

Rejected Offers

Admin Delivery Manager Talent Partner

Logs offers that were not accepted, recording the candidate name, role, and rejection reason. Reason categories are: Salary, Motivations, Counter-offer, Took another opportunity, and Other.

Filtering

Filter by project using the dropdown.

Recording a rejection

Use + Record Rejection to log a declined offer. This data feeds the Offer Rejection Reasons panel on the Project Dashboard, helping identify trends over time.

Admin Panel

Admin

The Admin Panel is visible to Admins only. It provides two management functions:

User Assignments

Lists all registered users, their email addresses, assigned role, and project assignments. Use this to grant or change a user's role, or to assign them to one or more projects. Users are auto-registered on first login — you can then update their role and project access here.

Project Management

Provides a full list of all projects with the ability to edit details, update status, and manage project records outside of the main Projects page.

If a user reports they can't see any data after signing in, check their user assignment here — they may be registered but not yet assigned a role or project.
